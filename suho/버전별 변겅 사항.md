## v2
v1 → v2 변경 요약
CatBoost cat_features 네이티브 활용이 가장 큰 변화입니다. 
v1에서는 카테고리 20개를 전부 LabelEncoding으로 숫자로 바꿔서 넣었는데, 이러면 CatBoost가 그걸 연속형 숫자로 인식합니다. 
v2에서는 문자열 그대로 넣고 cat_features 인덱스를 지정해서 CatBoost의 ordered target encoding이 작동하도록 했습니다. 
이 방식이 특히 카디널리티가 높은 카테고리(시술 유형, 배란 유도 유형 등)에서 효과가 큽니다.

하이퍼파라미터를 4개 후보 중에서 자동 선택합니다. depth 6/7/8, learning_rate 0.01/0.02/0.03 조합을 3-fold로 빠르게 비교한 뒤 가장 좋은 설정으로 본 학습을 진행합니다. 탐색 시간은 약 5분 정도입니다.

Platt Scaling calibration을 추가했습니다. OOF 확률에 로지스틱 회귀를 한 번 더 적용해서 확률을 보정합니다. AUC가 올라가면 적용하고 아니면 자동으로 건너뜁니다.

### 현재 v3의 문제점
- 첫째, 시술 유형과 난자/정자 출처를 삭제한 것이 치명적입니다. step6에서 "시술 유형, 난자 출처, 정자 출처"가 object라서 삭제됐는데, EDA를 보면 시술 유형(IVF vs DI)은 성공률 26.2% vs 12.9%로 차이가 크고, 난자 출처는 기증 31.5% vs 본인 25.8% vs 알수없음 12.9%로 매우 유의미합니다. 이 세 컬럼을 삭제하지 말고 숫자로 변환해야 합니다.

- 둘째, scale_pos_weight=3이 오히려 해가 됩니다. v1에서 CatBoost는 별도 불균형 처리 없이 0.7403을 달성했는데, scale_pos_weight를 넣으면 양성 예측이 과다해져서 확률 분포가 왜곡됩니다. 실제로 v3의 test 예측 mean이 0.4588인데 실제 양성비율은 0.258입니다. ROC-AUC 평가에서는 순위만 중요하긴 하지만, 과도한 가중치는 학습 자체를 불안정하게 만듭니다.

- 셋째, 이식 배아 수 0개인 36,544건의 처리가 중요합니다. EDA에서 이식 배아 0개의 성공률이 0.1%입니다. 이건 대부분 배아 저장용이나 기증용 시술로 실제로 이식을 하지 않은 케이스입니다. 이걸 구분하는 피처가 필요합니다.

#### EDA 기반 개선 아이디어
1. 삭제된 3개 컬럼 복원 및 인코딩

시술 유형은 IVF=1, DI=0으로 이진 변환하면 됩니다. 난자 출처는 본인=0, 기증=1, 알수없음=2로, 정자 출처는 배우자=0, 기증=1, 미할당=2, 배우자및기증=3으로 순서형 인코딩합니다. 이 세 컬럼은 object 삭제 대상에서 제외해야 합니다.

2. 실제 이식 여부 플래그

이식 배아 수가 0인 경우는 사실상 임신 시도를 하지 않은 것이므로 실제이식여부 = (이식된 배아 수 > 0) 플래그를 만들면 모델이 이 패턴을 명확히 학습합니다.

3. 배아 생성 목적 단순화

EDA에서 "현재 시술용"은 27.4%, "기증용+현재시술용"은 38.0%인데, 나머지(배아저장용, 난자저장용, 기증용 등)는 전부 성공률 0~0.1%입니다. 이걸 "시술목적_이식있음"이라는 이진변수로 만들면 강력한 신호가 됩니다.

4. 특정 시술 유형에서 정보 추출

BLASTOCYST 포함 시술은 35~37% 성공률이고, 콜론(:)이 포함된 반복 시술(ICSI:ICSI 등)은 1% 성공률입니다. is_blastocyst 외에 is_repeat_procedure = ':'가 포함도 추가하면 좋겠습니다.

5. 난자×정자 출처 조합

EDA에서 "기증난자+배우자정자"가 31.6%, "본인난자+배우자정자"가 25.7%로 차이가 있습니다. 조합 피처를 만들면 개별 출처보다 예측력이 높을 수 있습니다.

6. 나이와 이식배아의 상호작용

나이 1(18-34)이면서 이식배아 1~2개인 경우가 성공률이 가장 높을 텐데, 이 상호작용을 나이x이식배아수로 만들면 비선형 관계를 잡는 데 도움이 됩니다.

7. DI 구조적 결측 활용

DI 시술은 29개 변수가 100% 결측인데, 이걸 0으로 채우면 IVF에서 실제로 0인 경우와 구분이 안 됩니다. 결측수 피처는 이미 있지만, 더 직접적으로 is_DI를 넣는 게 낫습니다.

8. scale_pos_weight 제거

1로 설정하거나 아예 빼고, 모델이 자연스럽게 확률을 학습하게 두는 편이 AUC에 유리합니다.

## v7
v6 → v7 핵심 변경사항
v4를 베이스로 복귀했습니다. DI -1 처리를 제거하고 결측은 원래대로 0으로 둡니다. 상관 기반 자동 제거도 하지 않고, 중요도 0인 7개 피처만 수동으로 제거합니다.

XGBoost와 CatBoost를 서로 다른 피처셋으로 학습합니다. XGBoost는 v4 전처리(수동인코딩 + TE + 효율성지표 + 나이TE)를 적용한 숫자 데이터를 받고, CatBoost는 원본 카테고리를 문자열 그대로 받아서 네이티브 ordered target encoding을 사용합니다. 두 모델이 데이터의 다른 측면을 학습하므로 블렌딩 효과가 극대화됩니다.

블렌딩 가중치를 0~1 구간에서 5% 단위로 탐색한 뒤, 최적점 주변에서 1% 단위로 미세 조정합니다. CatBoost가 단독으로 0.7403이었고 XGBoost가 0.7392였으므로, CB 가중치가 높을 것으로 예상되지만 블렌딩으로 둘 다보다 높아질 가능성이 있습니다.\

### v12
확인된 핵심 문제
문제 1 – CatBoost 단독 성능이 v1부터 고정: v1에서 원본 67개 피처 + CatBoost로 0.7403을 달성했고, 이후 파생변수를 9개(v8), 21개(v9), 47개(v10)까지 추가했지만 CB OOF AUC는 0.7398~0.7404 범위에서 벗어나지 않는다. CatBoost의 네이티브 카테고리 처리가 이미 수동 파생변수가 제공하는 정보를 내부적으로 학습하고 있어서 추가 피처가 노이즈로만 작용한다.

문제 2 – XGB와 CB 예측 상관이 0.99: v11에서 CB↔XGB 상관 0.9938, v12에서도 0.9938. 두 모델이 거의 동일한 예측을 생산하므로 블렌딩 이득이 0.0002~0.0003에 불과하다. 전처리가 다르더라도 같은 테이블 데이터에서 같은 상위 피처(실제이식여부, 이식된 배아 수, 배아 이식 경과일)가 지배하면 예측이 수렴한다.

문제 3 – OOF vs Test 갭: 팀원은 OOF 0.7404에서 test 0.7422를 달성했다. 우리는 OOF 0.7405~0.7406이지만 test가 0.7417 수준이다. 이는 OOF를 더 올리는 것보다 일반화/과적합 관리가 test 점수를 좌우함을 보여준다.

문제 4 – 피처 수 증가 시 성능 하락 패턴: v4(73개) 0.7392 → v5(95개) 0.7390 → v6(78개) 0.7377. v9(CB 77개) 0.7404 → v10(CB 95개) 0.7403. 피처를 늘릴 때마다 미세하게 하락하는 경향이 반복된다. 이 데이터에서 정보량의 한계가 약 60~70개 피처 수준에 있다.

문제 5 – 실제이식여부 의존도: XGB에서 이 피처 하나의 중요도가 0.32~0.49를 차지한다. 이 변수가 사실상 타겟의 필요조건(이식 안 하면 임신 불가)이므로, 모델이 이 변수에 과도하게 의존하면 나머지 피처의 미세한 차별력이 억제된다.

v13 핵심 설명

모델 학습은 v12의 CatBoost 3-seed 앙상블과 완전히 동일하게 전체 데이터에서 진행한다. 변경은 오직 제출 직전 한 단계뿐이다. test에서 이식된 배아 수 == 0인 행의 예측 확률을 min(원래 예측, clip_value)로 클리핑한다.
