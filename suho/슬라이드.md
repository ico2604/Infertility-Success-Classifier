# 슬라이드 8. 데이터 전처리
1. 전처리 목적
- 문자열 중심의 의료 데이터를 모델이 학습 가능한 형태로 변환
- 범주형 정보는 유지하고, 필요한 컬럼만 수치형 파생 변수 추가
- 결측치·표기 방식 차이를 통일해 데이터 일관성 확보
2. 주요 전처리 과정
- 기본 컬럼 정리
    - ID 컬럼 제거
    - 학습 시 타겟(임신 성공 여부) 제거
- 결측치 처리
    - 문자형(object) 컬럼 결측치 → "Unknown"으로 통일
    - 수치형 컬럼 결측치 → 0으로 처리
- 횟수형 문자열 숫자 변환
    - 총 시술 횟수
    - 클리닉 내 총 시술 횟수
    - IVF 시술 횟수
    - DI 시술 횟수
    - 총 임신 횟수
    - IVF/DI 임신 횟수
    - 총 출산 횟수
    - IVF/DI 출산 횟수
- 나이 구간 숫자 변환
    - 시술 당시 나이
    - 난자 기증자 나이
    - 정자 기증자 나이
- Boolean / Flag 컬럼 정규화(다양한 표기 방식을 0/1로 통일)
    - 배란 자극 여부
    - 단일 배아 이식 여부
    - PGS / PGD 시술 여부
    - 동결 배아 사용 여부
    - 불임 원인 관련 flag
- 범주형 처리
    - 원본 범주형 컬럼은 유지
    - CatBoost에 categorical feature로 직접 입력

# 슬라이드 9. 팀원 모델링(수호)
1. 사용한 모델 : CatBoost 기반 Expert Ensemble (V21)
2. 구성 : Global base + Transfer expert+ Frozen expert
    - Global base: 기존 최고 조합인 v17-v19 blend 사용
    - Transfer expert : 학습 대상: 이식된 배아 수 > 0
    - Frozen expert : 학습 대상: 이식된 배아 수 > 0 AND 동결 배아 사용 여부 > 0
3. 핵심 피처
	- Transfer Expert에서 중요했던 피처
		- effective_oocyte_age(실제 사용 난자의 생물학적 연령) : 난자 기증자 나이_num
		- advanced_age_autologous_flag(고령 + 자가난자 사용 여부) : 
        - 시술 시기 코드
        - transfer_intensity(이식 배아 수 / 나이)
        - transfer_day_cat(배아 이식 시점)
        - 배아저장비율
        - freeze_to_transfer_ratio_domain(저장 배아 대비 실제 이식 전략)
    - Frozen Expert에서 중요했던 피처
        - 시술 시기 코드
        - thaw_to_transfer_ratio(해동된 배아 중 실제 이식까지 이어진 비율)
        - thaw_survival_proxy(해동 후 생존/활용 proxy)
        - transferable_embryos(이식 + 저장 가능한 배아 수)
        - effective_oocyte_age
        - ivf_failure_x_effective_oocyte_age(반복 실패 이력 × 난자 연령)
4. 인사이트
    - 전체 모델보다 subgroup expert가 더 유효
        - 전체 모델은 이미 포화 상태였고, 실제 개선 포인트는 이식군 / 동결이식군에 있었음
    - 도메인 피처는 global보다 expert에서 더 잘 작동
        - 난자 연령, 해동 생존성, transferable embryo 수 같은 피처는 전체 모델보다 transfer/frozen expert에서 더 높은 중요도를 보임
    - Frozen transfer는 별도 모델이 필요
        - frozen group에서는 transfer expert보다 frozen expert가 더 유효한 가중치를 가짐
        - 즉 동결 이식은 일반 이식과 다른 패턴을 가짐
    - 최종 기여
        - V21 OOF AUC: 0.740895 / Public LB: 0.7423956072
5. 발표 한 줄 요약 
    - v21은 전체 환자를 보는 Global 모델 위에, 이식 환자와 동결이식 환자 전용 Expert 모델을 추가한 구조로, frozen transfer 구간을 보완해 최종 팀 앙상블 성능 향상에 기여했다.

# 슬라이드 10. 팀원 모델링(소윤)
1. 사용한 모델 : Tree-based Rank Ensemble (v4)
2. 구성 : XGBoost v2 (reg_relax) + CatBoost v2 + LightGBM v1
    - Ensemble 방식 : Rank Ensemble (Percentile Rank 변환 후 Weighted 결합)
3. 핵심 피처
    - 전체 모델에서 중요했던 피처
        - 이식된 배아 수
        - 이식배아수 구간
        - 저장 대비 이식 비율
        - 배아 이식 경과일
        - 배아이식비율 / 배아저장비율
        - 총 생성 배아 수
    - 특징 : 배아 관련 정보와 이식 결과를ㅌ 직접적으로 반영하는 변수
4. 주요 파생 피처
    - 배아생성효율
    - ICSI 수정 효율
    - 난자 활용률
    - 전체임신률 / IVF임신률
    - 임신 유지율
    - 복합 위험도 점수
    - 특징 : 환자 상태와 시술 결과를 요약한 지표
5. 핵심 인사이트
    - Feature는 확장이 아니라 “선별”이 중요
    - Feature 추가 실험(v3~v5) → 성능 하락 -> 정보 밀도가 높은 feature 중심 구성이 효과적
    - 단일 모델 대비 Ensemble에서 성능 상승 -> 모델 간 서로 다른 패턴 결합이 중요
    - Rank Ensemble이 가장 안정적인 구조
    - Probability 방식 대비 안정적
    - 복잡한 구조보다 Rank 방식이 더 안정적-> 과적합 없이 일관된 예측 가능
    - 최적 weight: XGB 0.33 / CAT 0.41 / LGB 0.26-> 모델별 기여를 반영한 조합이 중요
    - 최종 기여 (v4 기준) 
        - OOF ROC-AUC: 0.740448 / Public LB: 0.74217
6. 발표 한 줄 요약
    - “핵심 feature를 기반으로 여러 모델을 Rank 방식으로 결합하여, 분포 차이에 강한 안정적인 예측 구조를 구성했습니다.”

# 슬라이드 11. 팀원 모델링(관주)
1. 사용한 모델 : 2-model Rank Ensemble
2. 구성 :  (XGB reg_relax + CatBoost v17 + LGB v1) + 개인 CatBoost 개선 모델
    - Ensemble 방식 : Percentile Rank 변환 후 비균등 가중 결합
    - 목적 : 단일 모델 성능 자체보다 일반화가 강한 rank 구조를 만드는 데 초점
2. 개인 모델 쪽 핵심 변경
    - CatBoost 5-seed 기반
    - NaN 보존
    - border_count 확대
    - 일부 노이즈 피처 제거
    - 신규 보조 피처 추가
    - age_failseverity
    - 배아이식일있음
    - 배아해동일있음
    - 임신경과연수있음
3. 핵심 피처
    - 전체 구조에서 중요했던 피처
        - age_transfer_interaction
        - transferintensity
        - 이식된 배아 수
        - 실제이식여부
        - 이식배아수구간
        - 배아 이식 경과일
        - transfer_day_cat
        - 배아저장비율
        - egg_spermcombo
        - 난자 출처
    - 개인 모델에서 의미 있었던 보조 피처
        - 배아이식일있음
        - age_fail_severity
4. 핵심 인사이트
    - 단일 모델 개선보다 강한 기존 앵커와의 rank 결합이 더 유효
    - 단일 CatBoost만으로도 성능은 높았지만, 최종적으로는 비균등 2-rank 구조가 public LB에서 더 안정적으로 작동함
    - OOF 최적화와 LB 최적화는 같지 않음
    - 내부 OOF가 약간 더 높다고 해서 LB가 반드시 더 좋아지지는 않았고, rank 기반 결합이 분포 차이에 더 안정적이었음
    - 개인 모델의 역할은 “새로운 메인 구조”보다 보완적 signal 제공
    - 결측 처리 방식, seed averaging, 보조 피처 추가를 통해 기존 강한 CatBoost 계열에 다른 예측 패턴을 제공함
    - 복잡한 확장보다 정제된 결합 구조가 더 중요
    - 대규모 피처 확장보다, 강한 모델 2개를 rank 방식으로 안정적으로 결합하는 쪽이 더 효과적이었음
    - 최종 기여(2-model rank)
        - OOF ROC-AUC: 0.740719 / Public LB: 0.7422826355

- Ensemble Insight (v21 + Rank Ensemble)

    - 본 실험에서는 v17과 v19 모델을 블렌딩한 CatBoost 기반 v21 모델을 중심으로,
    XGBoost와 LightGBM을 결합한 Rank Ensemble을 적용하였다.

- 인사이트 1 - 전체 모델은 이미 포화 상태였다
    - OOF AUC는 0.7390~0.7409 수준에서 거의 변하지 않았다.
    - 즉 단순히 피처를 더 넣는 방식만으로는 큰 개선을 만들기 어려웠다.

- 인사이트 2 - “강한 단일 모델 + 보정 모델 구조”
    - v21이 전체 예측의 중심을 담당하고,
    - XGB와 LGB는 보정 역할을 수행하는 구조로 작동하였다.
    - v21: 전체 예측 방향 결정 (80%)
    - XGB: 수치형 interaction 강화 → 경계 보정
    - LGB: 분기 기반 threshold 조정 → 미세 보정
    - 단일 모델을 대체하는 것이 아니라,
    - 강한 베이스 모델 위에 보정 모델을 얹는 전략이 효과적이었음

- 인사이트 3 - Rank Ensemble의 안정화 효과
    - Probability 평균 대비 Rank 기반 결합이 더 안정적인 성능을 보였다.
    - 모델 간 확률 스케일 차이 제거
    - extreme prediction 영향 감소
    - 모델 간 편향 완화
    - 결과적으로일반화 성능을 개선하는 방향으로 작용

- 인사이트 4 - 모델 다양성이 성능 향상에 기여
    - 낮은 가중치(XGB 16%, LGB 4%)의 모델도 의미 있는 기여를 했다.
    - 서로 다른 분할 방식
    - 다른 feature interaction 학습, 이는 앙상블에서 모델 다양성이 중요함을 보여주는 결과

- 인사이트 5 - Rank Ensemble은 “최종 polishing 단계”
    - Rank Ensemble은 성능을 크게 바꾸기보다, 이미 좋은 모델의 예측을 정교하게 다듬는 역할을 수행하였다.
    - 즉, Feature Engineering → 성능 기반
    - v21 모델 → 핵심 성능 확보
    - Rank Ensemble → 최종 성능 미세 조정

- 인사이트 6 - 개선해야 할 핵심 구간은 transfer / frozen transfer
    - subgroup 분석 결과, non-transfer는 이미 매우 높은 AUC를 보였고, 실제 병목은 transfer, 특히 frozen transfer였다.
    - 따라서 v21에서는 전체 모델 개선보다 subgroup별 expert 모델을 도입하는 것이 합리적이었다.

- 최종 결론
    - v17과 v19를 결합한 v21 모델을 중심으로, 서로 다른 특성을 가진 XGBoost와 LightGBM을 보조적으로 결합하고 Rank Ensemble을 적용함으로써, 모델 간 스케일 차이를 제거하고 예측의 안정성을 확보하여 최종 성능을 향상시켰다.

# 슬라이드 14. 회고 및 배운점
- 성능 향상의 핵심은 알고리즘이 아니라 도메인 기반 피처 설계였다.
- 모델과 피처는 함께 작용할 때 시너지 효과 발생 — 단일 요소 개선만으로는 한계
- Rank Ensemble 적용으로 예측 안정성 및 일반화 성능 확보.

- 도메인 기반 Feature Engineering — 의료 데이터에서는 의미 있는 파생 변수가 핵심
- 강한 단일 모델이 성능의 기반
    - 앙상블 전에 베이스 모델 완성도가 먼저 앙상블 = 예측을 안정화하는 마무리 단계 — 성능 향상보다 안정화에 집중.
- 모델 다양성이 성능 향상에 기여 — CatBoost, XGBoost, LightGBM의 상호 보완
